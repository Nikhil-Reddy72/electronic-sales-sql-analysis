# Electronic Sales Data Analysis (SQL Project)

## Project Overview
This project analyzes electronic sales transaction data to identify patterns in revenue generation, customer purchasing behavior, and product performance.

The analysis aims to generate business insights that help improve sales strategies, optimize marketing targeting, and enhance inventory planning.

## Tools Used
- MySQL
- SQL
- Data Analysis

## Dataset
The dataset contains electronic sales transactions including:

- Customer demographics
- Product details
- Order information
- Payment methods
- Shipping types
- Customer ratings

## Project Workflow

1. Database creation
2. Table creation
3. Data cleaning
4. Data quality checks
5. Exploratory data analysis
6. KPI calculations
7. Business insights generation

## Key SQL Analysis

- Revenue by product category
- Top selling products
- Customer segmentation
- Payment method analysis
- Shipping preference analysis
- Monthly sales trends
- Add-on purchase analysis

## Business Insights

- Few product categories contribute the majority of total revenue.
- Loyalty members spend more compared to non-loyal customers.
- Products with higher ratings tend to have higher sales.
- Add-on purchases increase the overall order value.
- Sales show seasonal demand patterns.

## Business Recommendations

- Focus marketing campaigns on top-performing product categories.
- Promote loyalty programs to increase repeat purchases.
- Highlight highly rated products in promotions.
- Use cross-selling strategies for add-on products.
- Introduce bundle offers to increase order value.
